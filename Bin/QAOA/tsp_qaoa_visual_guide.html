🎯 Travelling Salesman Problem com QUBO e QAOA

Modelagem, teoria e implementação de algoritmos quânticos para otimização combinatória

1. Introdução ao Problema

Travelling Salesman Problem (TSP): Encontrar o circuito mais curto que visita cada cidade exatamente uma vez e retorna à origem.

Complexidade

NP-Completo

Cresce exponencialmente com o número de cidades

Aplicações Práticas

Logística

Roteamento, delivery, design de circuitos

Por que Algoritmos Quânticos?

  • Exploração paralela do espaço de soluções
  • Potencial para speedup exponencial em certos problemas
  • QAOA executa em computadores quânticos reais (NISQ)
  • Viável para TSP pequeno-médio (4-8 cidades)

2. Formulação Matemática: QUBO

2.1 Definição do Problema

Seja uma instância TSP com:

  • n: número de cidades
  • d_ij: distância entre cidades i e j
  • x_it: variável binária (1 se cidade i visitada na posição t)

2.2 Função Objetivo

Distância total do circuito: C = Σ_i Σ_j Σ_t d_ij * x_it * x_j(t+1)

2.3 Restrições

Restrição 1: Cada cidade visitada exatamente uma vez

Σ_t x_it = 1, para todo i

Restrição 2: Uma cidade por posição

Σ_i x_it = 1, para todo t

2.4 Função QUBO Completa

H = H_custo + λ₁ * H_rest1 + λ₂ * H_rest2 Onde: - H_custo = Σ_i Σ_j Σ_t d_ij * x_it * x_j(t+1) - H_rest1 = Σ_i (1 - Σ_t x_it)² - H_rest2 = Σ_t (1 - Σ_i x_it)² - λ₁, λ₂: pesos de penalidade (típico: 10-100)

def build_qubo(distances, n_cities, lambda1=10, lambda2=10): Q = {} # 1. Termo de custo (distância) for i, j, t in indices: var1 = i * n + t var2 = j * n + (t + 1) % n Q[(var1, var2)] += distances[i][j] # 2. Penalidade: visitação única for i, t1, t2 in indices: var1 = i * n + t1 var2 = i * n + t2 Q[(var1, var2)] -= 2 * lambda1 # 3. Penalidade: posição única for t, i1, i2 in indices: var1 = i1 * n + t var2 = i2 * n + t Q[(var1, var2)] -= 2 * lambda2 return Q

4. QAOA: Quantum Approximate Optimization Algorithm

4.1 Conceito Básico

QAOA combina dois componentes:

🔴 Hamiltoniano de Custo (H_C)

Codifica a função objetivo (problema QUBO)

H_C = Σ_(i,j) Q[i,j] Z_i Z_j

🔵 Hamiltoniano de Mistura (H_M)

Explora o espaço de soluções com rotações X

H_M = Σ_i X_i

4.2 Estrutura do Circuito (p=2)

Inicialização:
q[0]: |0⟩ ─ H ─┬──────────────┬──────────────┬─ M
               │              │              │
q[1]: |0⟩ ─ H ─┤  Bloco 1     │  Bloco 2     │─ M
               │  (γ₁, β₁)    │  (γ₂, β₂)    │
q[n]: |0⟩ ─ H ─┴──────────────┴──────────────┴─ M

Bloco p (para p = 1, 2, ...):
  1. Aplicar H_C: e^(-i γ_p H_C)  [CZ gates + RZ]
  2. Aplicar H_M: e^(-i β_p H_M)  [RX gates]

4.3 Hiperparâmetros QAOA

Parâmetro Significado Range Típico Impacto
p Profundidade (níveis) [1, 5] Maior p → melhor aproximação
γ_p Ângulo Hamiltoniano custo [0, 2π] Controla evolução de custo
β_p Ângulo Hamiltoniano mistura [0, π] Controla exploração
shots Medições por iteração [100, 10000] Mais shots → menor ruído

5. Circuito Quântico Detalhado

5.1 Portas Lógicas Utilizadas

Porta Símbolo Descrição Uso em QAOA
Hadamard H Cria superposição (|0⟩ → (|0⟩+|1⟩)/√2) Inicialização
Pauli Z Z Aplica fase (|0⟩ → |0⟩, |1⟩ → -|1⟩) Operador do Hamiltoniano
Pauli X X Flip bit (|0⟩ ↔ |1⟩) Hamiltoniano mistura
Controlled-Z CZ Aplica Z em q[j] se q[i] é |1⟩ Entrelação (Hamiltoniano custo)
RX(θ) RX Rotação em torno eixo X por ângulo θ Mistura (aplicar H_M)
RZ(θ) RZ Rotação em torno eixo Z por ângulo θ Custo (aplicar H_C)

5.2 Bloco de Custo (Implementação)

Para cada termo Q[i,j] * Z_i * Z_j na matriz QUBO:

Bloco de custo com CZ:
i: ──RZ(2*Q[i,j])─●─
                   │
j: ─────────────●─

Equivalentemente com gates naturais:
i: ───●─────────
      │
j: ─RZ(2*γ)─●─

5.3 Bloco de Mistura

Aplicado em paralelo a todos os qubits:

Bloco de mistura:
q[0]: ─RX(2*β)─
q[1]: ─RX(2*β)─
...
q[n]: ─RX(2*β)─

(Todos os qubits aplicados simultaneamente)

8. Vantagens e Limitações

✓ Vantagens da Abordagem Quântica

  • Executa em computadores quânticos reais (NISQ)
  • Explora espaço de soluções em paralelo
  • Potencial para speedup em problemas grandes
  • Mais robusto que métodos exatos para instâncias grandes
  • Escalável para múltiplos qubits

✗ Limitações Atuais

  • Ruído quântico limita profundidade (p ≤ 3)
  • Difícil encontrar bons parâmetros iniciais
  • Requer n² qubits para n cidades
  • Métodos clássicos ainda melhores para TSP pequeno
  • Plateau barren (dificuldade de otimização)
⚠️ Consideração Importante: Para TSP com 4 cidades, métodos clássicos como força bruta ou branch-and-bound são preferíveis. QAOA é mais vantajoso para problemas com 10+ cidades e em computadores quânticos com suficiente coerência.

9. Extensões e Melhorias

VQE (Variational Quantum Eigensolver)

Alternativa mais robusta que QAOA para encontrar autovalores mínimos. Melhor para ruído quântico atual.

WARM START QAOA

Inicializar parâmetros com solução clássica boa em vez de aleatória. Reduz tempo de convergência.

ADAPTIVE QAOA

Ajustar p dinamicamente durante otimização. Adiciona novos níveis quando útil.

Constraint Encoding Aprimorado

Usar ansatz mais sofisticado que respeita automaticamente restrições. Reduz espaço de busca.

Hybrid Algorithms

Combinar clássico (pré-processamento) + quântico (otimização) + clássico (refinamento).
